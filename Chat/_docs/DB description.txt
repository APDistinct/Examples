Пользователи (в широком смысле) деляться на два типа (может три):
1) Непосредственно сами пользователи в узком смысле, т.е. те консультанты, кто использует програмное обеспечение.
2) Клиенты, которые взаимодействую с ПО опосредовано - общаются с помощью других мессенджеров с консультантами через офф аккаунты
3) Другие консультанты, не использующие ПО?

Все пользователи сводятся в одну таблицу User.User с полями: 
[Id]*, 
[Enabled] - признак, что не удалён
[FullName] (видимое имя), 
[IsConsultant], 
[RegistrationDate] (дата регистрации из ПАРТНЁРа),
[InsertDate] (дата вставки записи)
[SignUpDate] (дата начала использования ПО)
[Phone], 
[Email],
[Login], 
[PswHash], 
[OwnerUserId] (идентификатор консультанта, которому принадлежит клиент)
[DefaultTransportTypeId] - идентифкатор предпочитаемого транспорта
[AvatarUploadDate] - время загрузки аватара, null - если нет аватара
... <другие поля>

С пользователями связана таблица доступного для пользователя транспорта User.Transport:
[UserId]* (связь с таблицей User.Users 1-м),
[TransportTypeId] (тип транспорта WhatsUpp, Telegram,....),
[TransportOuterId] (идентификатор в транспорте),
[Enabled]

Таблица для аваторов[Usr].[UserAvatar]. Если есть запись, значит есть и аватара
[UserId]* - ссылка на пользователя (1-1)
[Data] - блоб поле

Контакты пользователя [Usr].[Contact]:
[UserId]* （владелец контакта - ссылка на User.Users),
[ContactId]* (контакт - ссылка на User.Users)

Сообщения [Msg].[Message]:
[Id]*,
[PostTm] - дата вставки
[MessageTypeId] (0 - персональное, 1 - в группу, 2 - рассылка),
[FromUserId] - отправитель
[FromTransportTypeId] - транспорт отправителя
[AnswerToId], (ответ на сообщение?)
[Text]
[IsDeleted] - признак удаления сообщения

Адресаты сообщения [Msg].[MessageToUser] 
[MsgId]* - сообщение
[ToUserId]* - кому
[ToTransportTypeId]* - куда
[IsFailed] - сбой
[IsSent] - отправлено
[IsDelivered] - доставлено
[IsRead] - прочитано

События [Msg].[Event]:
[Id]* (bigint),
[Tm],
[EventTypeId] - тип сообщения из справочника
[CausedByUserId] - кто явился причиной сообщения
[MsgId] (связанное сообщение, если есть)

Адресаты событий [Msg].[EventAddressee]
[Id] - идентификатор события
[UserId], (кому адресовано событие)

Информация о номере последнего доставленного уведомления Msg.EventDelivered:
[UserId],
[LastEventId]

При запросе событий указывается последний идентификатор + 1, с которого начинать выдачу, или если не указан, то вовзращаются ещё не отданные пользователю уведомления
